The hierarchical file system — invented in the 1960s — is still the dominant paradigm for data storage. This prototype replaces it with a relational, NLP-powered layer where files are discovered through natural language, not memorized directory paths.

Save a file anywhere. Find anything, instantly, by describing it.

How It Works

  • STEP 01
    On-Save NLP ScanWhen a file is saved, a local NLP model extracts entities: project names, dates, people, document types, topics, and relationships to other files.
  • STEP 02
    Tag Graph ConstructionTags are stored in a relational graph database, not a flat folder. Files can belong to multiple contexts simultaneously without duplication.
  • STEP 03
    Natural Language QueriesUser queries are parsed by the same NLP model, matched against the tag graph using semantic similarity, and ranked by relevance + recency.
  • STEP 04
    Manual Override & LearningUsers can add/remove tags manually. The system learns from corrections, improving tag accuracy over time per-user and per-domain.

Use Cases

Enterprise Knowledge Management
Large organizations accumulate massive, disorganized file stores. Semantic retrieval surfaces relevant documents regardless of who saved them or where.
Legal & Compliance Workflows
Law firms can retrieve all documents related to a case, party, or date range in seconds — without relying on rigid naming conventions.
Personal Productivity
Stop remembering folder paths. Describe what you need in plain language and find it instantly — even files you haven't touched in years.
Research & Academic Archives
Papers, datasets, and notes become a queryable knowledge graph — discoverable by concept, author, methodology, or date range.
